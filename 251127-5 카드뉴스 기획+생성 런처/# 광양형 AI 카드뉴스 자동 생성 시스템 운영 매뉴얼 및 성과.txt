# 광양형 AI 카드뉴스 자동 생성 시스템 운영 매뉴얼 및 성과 보고서

본 문서는 광양시 AI 활용연구 동호회에서 구축한 **'AI 기반 카드뉴스 자동화 시스템'** 의 구축 과정, 작동 원리, 사용법 및 예산 효율성을 상세히 기술합니다.

---

### 광양형 AI 카드뉴스 자동 생성 시스템 구축 개요

* **목표:** 텍스트(공고문)만 입력하면 기획부터 디자인까지 완성된 '광양시 스타일'의 카드뉴스 세트를 즉시 생성하는 100% 자동화 시스템 구축
* **기초 데이터:** 광양시 SNS에 게재된 카드뉴스 1,600여 장
* **작동 원리:** AI가 공고문 내용과 가장 어울리는 과거 우수 사례(레퍼런스)를 DB에서 찾아내고, 그 디자인 스타일을 모방하여 새로운 이미지로 생성
* **사용법:** 전용 런처 프로그램에서 공고문 파일(TXT/PDF)을 선택하고 '기획 시작' 및 '디자인 생성' 버튼을 차례로 누르면 완료
* **소요 예산:** 초기 DB 구축비 약 5,500원(1회성), 운영비 1세트(4장 기준)당 약 750원 수준으로 외주 대비 압도적인 예산 절감 효과
* **사용 툴:** Google Antigravity, Cursor, Claude Code (노코드 바이브 코딩 툴)
* **개발 기간:** 1일 (이미지 분류 및 전처리 포함)

## 1. 프로젝트 개요 및 데이터 자산화

본 프로젝트의 핵심은 **"기존에 축적된 우수한 홍보 자산을 AI의 두뇌로 이식하는 것"** 입니다.

### 1.1. 데이터 리소스 (Resource)
* **원본 데이터:** 광양시 SNS에 게재된 **총 1,600여 장의 광양시 기존 카드뉴스 이미지**.
* **데이터의 가치:** 단순한 이미지 파일이 아니라, 광양시만의 디자인 톤앤매너, 시민 친화적 문체, 정보 전달 방식이 담긴 '홍보 DNA'의 집약체입니다.

### 1.2. AI 기반 자동 분류 및 DB 구축 (핵심 성과)
1,600장의 데이터를 사람이 일일이 열어보고 분류하는 것은 비효율적입니다. 저희는 이 과정을 **100% AI 자동화 기술**로 해결했습니다.

* **자동 분류 시스템:** Python 코드를 통해 **Google Gemini Vision API**를 호출, AI가 1,600장의 이미지를 1장씩 보면서 **"이건 표지(COVER)", "이건 본문(BODY)", "이건 마무리(OUTRO)"** 라고 판단하고 폴더를 자동 정리하게 했습니다.
* **심층 분석 및 DB화:** 분류된 이미지를 다시 AI가 분석하여 **'분위기(활기찬, 진지한)', '색감', '텍스트 배치 방식'** 등의 메타데이터를 추출, 검색 가능한 **'스타일 색인표(Index DB)'** 를 구축했습니다.
* 본 과정에서 **추출된 카드뉴스 문안과 구조를 활용** 하여 공고문/보도자료 등 원문 텍스트에서 **광양시 카드뉴스 스타일** 의 문안을 자동 작성하도록 하였습니다.

## 2. 시스템 작동 원리 (Mechanism)

이 시스템은 **RAG (Retrieval-Augmented Generation, 검색 증강 생성)** 기술과 구글의 최신 언어 모델(Gemini-2.5-Pro) 및 이미지 생성 모델(Gemini-3.0-Pro-Image-Preview, 일명 Nano Banana Pro)을 기반으로 작동합니다.

1.  **입력 (Input):** 담당자가 공고문(TXT/PDF)을 넣습니다.
2.  **검색 (Retrieval):** AI가 공고문의 내용을 분석한 뒤, 구축된 **'스타일 색인표'**에서 해당 내용과 가장 잘 어울리는 **'과거의 우수 사례(레퍼런스)'** 를 찾아냅니다. (예: 청년 정책 공고문 -> '활기찬 청년 지원사업' 스타일 검색)
3.  **기획 (Planning):** 찾아낸 우수 사례의 문체를 참고하여, 공고문을 **카드뉴스용 멘트(제목, 요약)** 로 자동 변환합니다.
4.  **생성 (Generation):** 확정된 멘트와 레퍼런스 이미지를 **Gemini 3 Pro(최신 멀티모달 모델)** 에게 전달하면, AI가 레퍼런스의 디자인을 모방하여 새로운 이미지를 그려냅니다.

    ※ 전체 이미지를 전부 제공하지 않고, 사전 구축된 인덱스를 활용하여 이미지 일관성과 API 비용 절감을 동시에 실현)

## 3. 상세 사용법 (User Guide)

통합 런처(`launcher.py`)를 실행하여 **원클릭**으로 진행합니다.

1.  **[Step 1] 기획안 생성:**
    * `📂 파일 열기` 버튼을 눌러 공고문 파일(TXT 또는 표가 포함된 PDF)을 선택합니다.
    * `✨ AI 기획 시작` 버튼을 누르면 AI가 문안을 작성해줍니다.
    * *(옵션)* `📝 기획안 수정` 버튼을 눌러 AI가 쓴 제목이나 내용을 다듬을 수 있습니다.
2.  **[Step 2] 디자인 생성:**
    * 원하는 **비율(4:5, 1:1, 9:16)** 을 선택합니다.
    * `🎨 AI 디자인 생성 시작` 버튼을 누르면 끝입니다.
3. 몇 페이지부터 생성을 시작할지 선택하고, 생성된 이미지를 확인 후 계속 생성합니다.
4.  **[결과 확인]:** `완성된 카드뉴스` 폴더에 이미지가 자동 저장됩니다.
    * 중간에 오류가 발생하더라도 원하는 페이지부터 다시 시작할 수 있으며, 무조건 전체 페이지를 생성하는 것이 아니라 중간 결과물을 확인하고 다음 페이지 생성을 선택할 수 있는 방식으로 구축하였습니다.

## 5. 시스템의 차별점 및 우수성

### ✅ "일반 AI"가 아니라 "광양시 AI"입니다.
* ChatGPT나 Gemini 웹사이트에서 "카드뉴스 만들어줘"라고 하면 일반적이고 뻔한 디자인이 나옵니다.
* 본 시스템은 **우리 시가 실제로 만들었던 1,600장의 데이터를 학습(참조)** 하여 생성하므로, **광양시 특유의 친근한 말투, 선호하는 색감, '매돌이' 캐릭터 활용 규칙**을 정확히 따릅니다.

### ✅ 편리함과 생산성 극대화
* 웹에서 프롬프트(명령어)를 매번 길게 칠 필요가 없습니다. 파일만 넣고 클릭하면 끝나는 **GUI(윈도우 프로그램) 형태**로 제작되어 누구나 쉽게 사용할 수 있습니다.
* 특히 **PDF의 표(Table) 인식 기능**을 탑재하여, 복잡한 지원금액/일정 표도 깨지지 않고 정확하게 요약합니다.

## 6. 현재 기술의 한계점 및 대응

본 시스템은 구글의 최신 기술인 **`Gemini-3-pro-image-preview` (일명 Nano Banana Pro)** 모델을 사용합니다. 이에 따른 한계점과 대응책은 다음과 같습니다.

* **한계점:** 해당 모델은 현재 **'Preview(사전 공개)'** 단계의 최신 모델입니다. 전 세계적인 사용량 폭증으로 인해 구글 서버 응답에 종종 **병목(500 Error, 지연)** 이 발생할 수 있습니다.
* **대응책:** : 시스템 내부에 **'자동 재시도(Retry) 알고리즘'** 을 탑재하여, 에러 발생 시 스스로 복구를 시도합니다.

### 7.2. 인프라 구축 및 접근성 개선 (Infrastructure)
* **현황:** 보안상의 이유로 행정망(내부망) PC에서는 외부 API(Google Gemini) 직접 호출이 차단되어 있음.
* **과제:** 내부 직원들이 행정망 PC에서도 편리하게 시스템을 이용할 수 있는 환경 마련 필요.
* **조치 계획:**
    * **외부망 PC(또는 클라우드 서버)에 시스템 및 DB 호스팅:** 인터넷 접속이 가능한 외부망 환경에 Python 구동 서버를 구축.
    * **웹 애플리케이션(Web App) 전환:** 현재의 런처(EXE) 방식을 웹 사이트 형태로 전환하여, 내부망에서 URL 접속만으로 이용 가능하도록 개선 (단, 망연계 보안성 검토 필요).

### 7.3. 보안 조치 강화 (Security)
* **과제:** 웹 서비스 전환 시, 외부인의 무단 접속 및 데이터 유출 방지 대책 필요.
* **조치 계획:**
    * **접속 권한 통제:** 광양시청 내부 IP 대역에서만 접속 가능하도록 방화벽 설정(IP Whitelisting).
    * **사용자 인증:** 별도의 로그인 페이지를 구축하여 인가된 직원만 이용하도록 보안 조치 적용.

### 7.4. 생성 품질 고도화 및 미세 튜닝 (Quality Tuning)
* **현황:**
    * **환각 현상(Hallucination):** AI가 레퍼런스 이미지(과거 자료)의 텍스트(부서 연락처, 날짜 등)를 그대로 베껴와 잘못 기입하는 오류가 간혹 발생함.
    * **캐릭터 통제:** 강력한 금지 프롬프트에도 불구하고, 드물게 '매돌이'가 아닌 엉뚱한 캐릭터가 생성되기도 함.
* **대응 방안:**
    * **단기 (현재)**
        * 생성 속도가 빠르므로 **'N회 생성 후 선택'** 방식으로 운영 (잘못된 결과물은 폐기하고 다시 생성).
        * **제미나이 웹 또는 구글 AI 스튜디오**를 통해 개별 수정
        * **Photoshop, Affinity 등 이미지 편집 툴**을 사용하여 개별 수정
    * **중장기:**
        * **OCR 후처리 로직 개발:** 생성된 이미지의 텍스트를 다시 읽어와(OCR), 기획안의 연락처와 일치하는지 검증하는 자동 검수 모듈 도입 고려.
        * **파인 튜닝(Fine-tuning):** 향후 데이터가 더 축적되면, Gemini 모델 자체를 광양시 전용으로 미세 조정(Fine-tuning)하여 캐릭터 및 텍스트 오류율을 원천적으로 낮추는 방안 검토.

## 8. 시스템의 확장성 및 파급 효과 (Scalability & Impact)

본 시스템은 단순한 '카드뉴스 제작 툴'이 아닌, 데이터와 프롬프트 교체만으로 다양한 영역에 적용 가능한 **'가변형 AI 콘텐츠 생성 플랫폼'** 으로 설계되었습니다.

### 8.1. 홍보 매체의 무한 확장 (One-Source Multi-Use)
* **원리**
    * 현재의 로직(기획-검색-생성)은 유지하되, **학습 데이터(DB)의 종류와 프롬프트만 변경**하면 즉시 다른 형태의 결과물 생성이 가능합니다.
    * 충분한 데이터만 확보한다면 프롬프트 교체는 **비개발자도 노코드 바이브 코딩 방식으로 수행**할 수 있습니다.
* **적용 예시:**
    * **DB와 프롬프트를 '포스터'로 교체 시:** 행사 포스터, 배너 자동 디자인 시스템으로 전환.
    * **DB와 프롬프트를 'PPT'로 교체 시:** 보고서 슬라이드, 발표자료 자동 생성 시스템으로 전환.


### 8.2. 타 기관 확산 및 행정 혁신 선도 (Standardization)
* **표준 모델 제시:** 본 시스템은 특정 소프트웨어에 종속되지 않는 범용적인 파이썬(Python) 및 구글 API 기반으로 구축되어, **타 지자체나 공공기관에 즉시 '수평 전개'가 가능**합니다.
* **광양형 표준 전파:** 타 기관에서도 본 시스템의 구조를 가져가서 **'자신들의 데이터(기관 스타일)'** 만 갈아끼우면, 즉시 해당 기관의 스타일을 살린 전용 AI 이미지 생성 시스템을 구축할 수 있습니다.
* **의의:** 광양시가 단순 도입을 넘어, **지자체 AI 행정 혁신의 표준 모델을 제시하고 선도**한다는 점에서 큰 의의를 가집니다.
